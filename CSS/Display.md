To decyduje, jak element zachowuje się.

`display: block`
- zajmuje szerokość
- łamie linię
- możesz dać w/h
Np. `div, p, h1`

`display: inline`
- NIE łamie linii
- NIE przyjmuje w/h
- tylko content decyduje o rozmiarze
Np. `span, a`

`display: inline-block`
- nie łamie linii
- MOŻE mieć w/h
- idealny kompromis

`display: flex`
- układ w JEDNEJ osi
- centrowanie jak magia
- dopasowanie elementów

`display: grid`
- układ w DWÓCH osiach
- wielkie layouty
- dashboardy, galerie, sekcje
